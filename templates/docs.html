Useful APIs

Examples of the product’s backend capabilities

  • GET /api/dashboardJSON data powering the dashboard
  • POST /api/admin/appointmentsCreate a booking from the UI
  • POST /api/appointments/{id}/updateEdit appointment details and notes
  • POST /api/calls/{id}/lead-scoreScore a patient lead as hot, warm, or cold
  • POST /api/contact-requestCapture demo requests from the public landing page
  • GET /api/export/appointments.csvExport bookings to CSV

Current Free Feature Set

All of these work without paid AI usage

  • Persistent storageSQLite stores appointments, slots, calls, settings, and notes.
  • Search and filtersDedicated operational pages help staff review activity faster.
  • Edit and delete workflowsAppointments can be updated, annotated, or removed.
  • Lead prioritizationCalls can be categorized with free lead scoring logic.
  • Analytics and exportsProduct usage can be tracked without external analytics tools.
  • Calendar and patient viewsStaff can review upcoming days and repeat-patient history.
  • Demo request capture and audit logLanding-page leads and product changes are both tracked.
